Ecological Modelling and Data Analysis

My toolbox

A quick run-down on the software and techniques I regularly use when producing ecological models and working with large and complex data sets. I am always learning new skills and building on others. Technology is always changing and I want to be able to always upgrade to better systems and workflows. If there is something you feel is lacking from my skillset then please let me know via Twitter!

R + RStudio GIS (ArcGIS and QGIS) HTML, Plotly and JavaScript
R is an open source programming language and statistcal toolbox with an extensive library of packages for working with biological data. It is my go-to tool for manipulating, visualising and analysising data, in addition to producing ecological models. I use RStudio as a user interface for R to improve my workflow. Key packages I use are; ggplot2 and dplyr (also see this useful data wrangling cheat sheet) I use GIS (geographical imaging systems) to produce maps to illustrate and anlysis of spacial data. I have used ArcGIS for woking with marine spacial data in a module taught by Matt Witt which covered every stage of GIS data, from collection to visualisation to analysis. I have previously used the open source eqiuvalant software; QGIS. When producing maps I make sure to follow the comprehensive 'Graphics for Conservation' guide produced by Oryx. My motivation to learn HTML, the language of the web, is to be able to communicate my scientific ideas clearly and imformatively. Plotly is a way of easily producing graphs for online publishing, it is open source and connects well with R and ggplot2. I write my HTML in Brackets a clean, open source HTML editor. I am learning javascript as a way to make my web design more interative and engaging. Additionally, I try to bring together other widgets and tools that can be embedded, which you will find on throughout this website.

SQL Microsoft Office Other tools
A more recent addition to my skill set, having completed the Code Academy course on SQL I have been working with SQL databases with my work with COMPADRE plant matrix database. I'm hoping to integrate this into my web design, the software I use is MySQL. A staple of anyone who as worked on a computer, easy to use and intuiative. I've used Excel extensively for initial imputing of data, using formulas and VBA macros for calculations on-the-fly and for export to R. I have used Access for database work, used Powerpoint to produce clear and informative presentations and Word for report writing. MATLAB: The first programming language I learnt and used frequently in my mathematical modelling modules, an easy interface to work with.
Python: Another programming language I have experience with using.
Google Sheets/Forms: Simple and easy to use resources for collaborative projects, useful for backing up and sharing data, multi platform too.
GIMP: Open sources image manipulation software for a multitude of uses such as finalising graphs or producing figures.


Recent Projects and Courses

COMPADRE database project: An extensive research project using the COMPADRE plant matrix database. The database is an R object with 6242 entries and metadata. I produced a HTML readout of each entry in the matrix so that I could browse the database easier than through the R console or exporting to an Excel sheet. The project also had a literature review component looking at the original papers, I created an Access database to input my data from the literature review, but then decided an Excel spreadsheet was easier to use. I constructed mutiple models to test my hypothesis and analysed data in R. When the project is finished I will create a page dedicated to it's results with graphs produced in Plotly.

Marine Spacial Ecology (Third Year Module): A comprehensive 10 week course in ArcGIS, primarily dealing with marine data but with transferable skills such as making maps from data tables to shape files, spacial analysis. Including carrying out our own fieldwork and data collection.

Epidemic Modelling Module (Third Year Module): Used R to construct SIR epidemic modules, analysyse the output and experiment with implementing different disease preventation strategies such as vaccination. We then compared their effectiveness, this was all carried out in R using ggplot2 to visualise the data.

Mathematical Biology (Third Year Module): Using MATLAB to produce more mathematical models including; epidemic dynamics, optimal management modelling, game theory and evolution of cooperation. This module had a lot of pure mathematical content including; solving differential equation and equilibrium theory.

Tenerife Field Course: Two other students and I carried out a two day field study into invertebrate communities underneath rocks to see if the theory of island biogeography plays a role in this model system. Data was input into excel and imported into R for analysis. We also recorded gps coordinates of of the rocks allowing me to produce a GIS map of the area over satellite image of the area to illustrate the field site for my research poster.

Mathematics of the Environment (Second Year Module): Using MATLAB to produce mathematical models; climate modelling, epidemic modelling and population projection matrices. This model introduced me to the foundations of coding, a skill that has grown ever since. We wrote a total of four reports during the span of this module and were constantly coding and working on our MATLAB programs.

Iceland Field Course: An intensive week-long field course of data collection, analysis and presenting to the rest of the group. In a group of 11 we collected data and colaboratively input into a Google spreadsheet. I was in charge of data analysis in R and produced a number of graphs including an ordination plot of species dissimilarity between sites in geothermally heated streams, and those not heated. I used the vegan package in R to produce these plots.
